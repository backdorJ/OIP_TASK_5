Репозиторий содержит решение цепочки заданий:
1) скачивание документов,  
2) токенизация/лемматизация,  
3) инвертированный индекс и булев поиск,  
4) расчёт TF-IDF по терминам и леммам.

## Требования

- Python 3.9+
- зависимости из `requirements.txt`

Установка зависимостей (рекомендуется через virtualenv):

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Для каждого документа из `pages/` считаются:

- `TF(термин)` = число вхождений термина / общее число терминов в документе
- `IDF(термин)` = `log(N / df)`
- `TF(лемма)` = сумма вхождений всех терминов, приводящихся к лемме / общее число терминов в документе
- `IDF(лемма)` = `log(N / df)`

где:
- `N` — число документов,
- `df` — число документов, в которых встречается соответствующий термин/лемма.

### Запуск

```bash
python3 build_tfidf.py
```

### Результаты

- `tfidf_terms/<N>.txt` — TF-IDF по терминам, один файл на документ.
- `tfidf_lemmas/<N>.txt` — TF-IDF по леммам, один файл на документ.

Формат строки в каждом выходном файле:

```text
<термин> <idf> <tf-idf>
```

и

```text
<лемма> <idf> <tf-idf>
```

Каждый `<N>.txt` соответствует документу `pages/<N>.txt`.
